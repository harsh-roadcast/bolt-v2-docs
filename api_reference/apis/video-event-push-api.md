# Video Event Push API

### Overview

This document describes the payload structure used to push video or safety events from the server to the client API. The payload contains device information, location data, event metadata, and a URL reference to the recorded video or media.<br>

### **Authentication**

{% hint style="info" %}
Note: As this is a Push API, the client must verify incoming requests from the server.
{% endhint %}

This webhook push uses a standard Bearer Token or Basic Auth configured by the client.

```
Authorization: Bearer <client-provided-token>
```

### Endpoint

{% hint style="info" %}
_Configured by Client_
{% endhint %}

```
POST https://<client-domain.com>/api/webhook/video_events
```

### HTTP Method

```
POST
```

### Request Headers

| Header        | Value                                |
| ------------- | ------------------------------------ |
| Content-Type  | `application/json`                   |
| Authorization | Bearer `<token>` (Client configured) |

### **Sample Webhook Request**

This is an example of what the server will push to your endpoint:

```bash
curl --location 'https://<client-domain.com>/api/webhook/video_events' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer client_token_here' \
--data '{
    "Imei": "123456789012345",
    "FixTime": "2026-03-16 10:12:20",
    "ServerTime": "2026-03-16 10:12:25",
    "Url": "https://example.com/video.mp4",
    "Latitude": 28.6139,
    "Longitude": 77.2090,
    "DeviceID": "4587",
    "Address": "New Delhi, India",
    "Type": "video",
    "Alert": "fatigueDriving",
    "VehicleNo": "DL01AB1234"
}'
```


### Expected Client Response

To prevent the server from retrying the push, your endpoint must return a `200 OK` status.

```json
{
  "status": "success",
  "message": "Event received"
}
```

{% hint style="info" %}
**Note**: If the client endpoint returns a 4xx or 5xx error, the server will retry pushing the payload according to the configured retry policy.
{% endhint %}
